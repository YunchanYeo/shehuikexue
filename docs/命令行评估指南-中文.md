# 命令行评估指南（中文）

本文说明：**从零开始安装 → 标定常模（基准）→ 单条/批量评估 → 查看结果** 的完整命令。  
全部在本机运行，不调用云端大模型 API。

---

## 一、你需要准备什么

| 项目 | 要求 |
|------|------|
| 操作系统 | macOS（本文以 Mac 为例） |
| Python | 建议 **3.10–3.13**（不要用 3.14，部分依赖可能装不上） |
| 网络 | **首次运行**需联网（下载 Whisper 模型与 Python 包） |
| 录音文件 | 支持 `wav` / `mp3` / `m4a` / `flac` / `ogg` / `aac` / `wma` |
| 标定样本 | 至少 **2 条**录音（建议用同一批受试者录音，如 20–50 条） |

## 二、一次性安装（每个电脑只做一次）

打开 **终端**，进入项目目录（请改成你电脑上的实际路径）：

```bash
cd /Users/yeoyunchan/Documents/project/shehuikexue
```

创建并激活虚拟环境：

```bash

source .venv/bin/activate
```

安装依赖：

```bash

  -i https://pypi.tuna.tsinghua.edu.cn/simple \
```

说明旧虚拟环境已损坏，请删除后重建：

```bash

  -i https://pypi.tuna.tsinghua.edu.cn/simple \
  --trusted-host pypi.tuna.tsinghua.edu.cn
```


## 三、评估前的必做步骤：标定常模（基准）

### 为什么必须先做？

最终分数用 **Z 分** 计算：

```
Z = (该录音的指标值 − 群体均值) / 群体标准差
```

群体均值和标准差保存在 `speech_eval/norms.yaml`。  
**单条录音无法自己和自己比**（N=1 时标准差为 0），所以必须先用**多条录音**标定。

### 命令

把下面路径改成你的录音文件夹（示例：桌面上的 `录音` 文件夹）：

```bash
cd /Users/yeoyunchan/Documents/project/shehuikexue
source .venv/bin/activate

python calibrate_norms.py "/Users/yeoyunchan/Desktop/录音"
```

子文件夹里也有录音时，加 `-r`：

```bash
python calibrate_norms.py "/Users/yeoyunchan/Desktop/录音" -r
```

### 成功标志

终端末尾应出现类似：

```
已写入常模: .../speech_eval/norms.yaml
  样本数: 54
```

打开 `speech_eval/norms.yaml`，确认 `_meta.n_samples` ≥ 2，且各指标有 `mean` 和 `std`（不是占位值）。

### 可选参数

```bash
# 指定输出常模文件
python calibrate_norms.py "/path/to/录音" -o speech_eval/norms.yaml

# 使用更大的 Whisper 模型（更准但更慢）
python calibrate_norms.py "/path/to/录音" --whisper-model small

# 关闭降噪
python calibrate_norms.py "/path/to/录音" --no-denoise
```

## 四、正式评估

> **每次新开终端**，都要先执行：
>
> ```bash
> cd /Users/yeoyunchan/Documents/project/shehuikexue
> source .venv/bin/activate
> ```

### 4.1 单条录音评估（试跑推荐）

```bash
python evaluate.py "/Users/yeoyunchan/Desktop/录音/1.m4a"
```

保存文字报告：

```bash
python evaluate.py "/Users/yeoyunchan/Desktop/录音/1.m4a" -o 结果_1.txt
```

输出 JSON：

```bash
python evaluate.py "/Users/yeoyunchan/Desktop/录音/1.m4a" --json
```

已有转写文本、跳过语音识别时：

```bash
python evaluate.py "/path/to/1.m4a" --transcript "你好，我认为这个问题很重要。"
```

### 4.2 批量评估（推荐：整文件夹）

```bash
python evaluate_batch.py "/Users/yeoyunchan/Desktop/录音"
```

完成后，在**录音文件夹内**生成：

| 文件/文件夹 | 内容 |
|-------------|------|
| `评估汇总表.xlsx` | **推荐**：Excel 直接打开；含「评估汇总」「说明」两个工作表 |
| `评估汇总表.csv` | 同上数据的 CSV 备份 |
| `评估汇总说明.txt` | 按总分排序的可读汇总 + 指标说明 |
| `结果_json/` | 每条录音的详细 JSON（含转写、原始指标、Z 分） |

自定义输出路径：

```bash
python evaluate_batch.py "/Users/yeoyunchan/Desktop/录音" \
  --csv "/Users/yeoyunchan/Desktop/录音/我的汇总.csv" \
  -o "/Users/yeoyunchan/Desktop/录音/详细json" \
  --combined "/Users/yeoyunchan/Desktop/录音/全部结果.json"
```

### 4.3 可选参数（单条与批量通用）

```bash
--whisper-model small    # 识别更准，更慢（默认 base）
--device cuda            # 有 NVIDIA GPU 时用（Mac 一般用 cpu）
--no-denoise             # 关闭降噪
--denoise-strength 0.75  # 降噪强度 0.0–1.0
--norms speech_eval/norms.yaml   # 指定常模文件
```

## 六、计分公式（结果表对应关系）

| 维度 | 公式 |
|------|------|
| **流利度** | ¼×(Z发音速率 − Z无声停顿占比 − Z填充词密度 − Z重复/修正密度) |
| **丰富性** | ⅓×(Z词汇多样性 + Z平均句长 + Z连接词密度) |
| **生动性** | ½×(Z基频标准差 + Z强度标准差) |
| **总分** | (流利度 + 丰富性 + 生动性) ÷ 3 |

`评估汇总表.xlsx` / `评估汇总表.csv` 中「Z分·…」列即各指标的 Z 值；维度分列即加权后的综合分。

## 八、常见问题

### 1. 把示例路径 `/경로/녹음폴더` 原样粘贴了

那是占位符，必须改成真实路径，例如 `"/Users/yeoyunchan/Desktop/录音"`。

### 2. 一次粘贴了多行带 `#` 的注释

终端会把 `#` 开头行当命令执行。请**一行一行**复制运行。

### 3. 标定做了，但 Z 分看起来不对

确认评估时用的是**同一份** `speech_eval/norms.yaml`，且标定样本与待评样本属于同一群体（同一课程、同一任务）。

### 4. 54 个文件要跑多久？

取决于电脑性能，通常 **30 分钟～2 小时**。请勿让 Mac 休眠。

### 5. 换了一批新学生录音

应对**新批次**重新运行 `calibrate_norms.py`，再运行 `evaluate_batch.py`。
