# 명령줄 평가 사용 가이드（한국어）

이 문서는 **설치 → 기준치(常模) 보정 → 단건/일괄 평가 → 결과 확인** 까지 필요한 명령을 순서대로 정리합니다.  
모든 처리는 로컬에서 이루어지며, 클라우드 LLM API는 사용하지 않습니다.

## 1. 사전에 준비할 것

| 항목 | 요구 사항 |

## 3. 평가 전 필수: 기준치 보정（常模标定）

### 왜 필요한가?

최종 점수는 **Z-score**로 계산합니다:

```
Z = (해당 녹음의 지표값 − 집단 평균) / 집단 표준편차
```

집단 평균·표준편차는 `speech_eval/norms.yaml`에 저장됩니다.  
**녹음 1개만으로는 표준편차가 0**이 되어 Z점수를 만들 수 없으므로, 먼저 **여러 녹음**으로 기준을 맞춰야 합니다.

### 명령어

녹음 폴더 경로를 본인 환경에 맞게 바꿉니다（예: 바탕화면 `录音`）:

```bash
cd /Users/yeoyunchan/Documents/project/shehuikexue
source .venv/bin/activate

python calibrate_norms.py "/Users/yeoyunchan/Desktop/录音"
```

하위 폴더까지 포함:

```bash
python calibrate_norms.py "/Users/yeoyunchan/Desktop/录音" -r
```

### 성공 여부 확인

터미널 마지막에 비슷한 메시지가 나오면 성공:

```
已写入常模: .../speech_eval/norms.yaml
  样本数: 54
```

`speech_eval/norms.yaml`을 열어 `_meta.n_samples`가 2 이상이고, 각 지표에 `mean`/`std`가 채워져 있는지 확인합니다.

### 선택 옵션

```bash
# 상수 파일 경로 지정
python calibrate_norms.py "/path/to/录音" -o speech_eval/norms.yaml

# 더 큰 Whisper 모델（정확↑, 느림↑）
python calibrate_norms.py "/path/to/录音" --whisper-model small

# 노이즈 제거 끄기
python calibrate_norms.py "/path/to/录音" --no-denoise
```

```bash

--norms speech_eval/norms.yaml   # 상수 파일 지정
```

## 6. 점수 공식（결과표와의 대응）

| 차원 | 공식 |
|------|------|
| **流利度（유창성）** | ¼×(Z发音速率 − Z无声停顿占比 − Z填充词密度 − Z重复/修正密度) |
| **丰富性（풍부성）** | ⅓×(Z词汇多样性 + Z平均句长 + Z连接词密度) |
| **生动性（생동성）** | ½×(Z基频标准差 + Z强度标准差) |
| **总分（총점）** | (流利度 + 丰富性 + 生动性) ÷ 3 |

`评估汇总表.xlsx` / `评估汇总表.csv`의 「Z分·…」열 = 각 지표 Z값, 「流利度/丰富性/生动性/总分」열 = 가중 합산 점수.

## 8. 자주 하는 실수

### 예시 경로 `/경로/녹음폴더`를 그대로 붙여넣음

반드시 실제 경로로 바꿉니다. 예: `"/Users/yeoyunchan/Desktop/录音"`

### 주석(`#`)이 포함된 여러 줄을 한꺼번에 실행

`#`로 시작하는 줄은 셸이 명령으로 해석해 오류가 납니다. **한 줄씩** 실행하세요.

### 1단계(보정)만 하고 2단계(평가)를 안 함

`calibrate_norms.py`는 **기준선**만 만듭니다. 실제 점수표는 `evaluate_batch.py` 실행 후 `评估汇总表.xlsx`에서 확인합니다.

### 새 학생·새 과제 녹음을 추가했을 때

**새 집단**이면 `calibrate_norms.py`를 다시 돌린 뒤 `evaluate_batch.py`를 실행하세요.

### 54개 파일 소요 시간

보통 **30분~2시간**. Mac이 잠들지 않게 설정하세요.

## 9. 관련 파일

```
evaluate.py          # 단건 평가
evaluate_batch.py    # 일괄 평가
calibrate_norms.py   # 기준치 보정
speech_eval/norms.yaml   # 집단 mean/std（보정 결과）
app_gui.py           # GUI（단건）
```

### 결과 해석 참고

- **发音速率**: 무음 제거 후 초당 음절 수（높을수록 말이 빠름）
- **无声停顿占比**: 0.2초 이상 무음 비율（높을수록 망설임）
- **填充词密度**: 嗯/啊/就是/那个 等 비율（높을수록 유창성↓）
- **词汇多样性(TTR)**: 고유 단어 / 전체 단어（높을수록 어휘 풍부）
- **基频标准差 / 强度标准差**: 억양·음량 변화（클수록 생동감↑）

자세한 중국어 설명은 `评估汇总说明.txt`에도 포함됩니다.
